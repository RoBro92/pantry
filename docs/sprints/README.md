# Sprint Notes

This directory holds the public implementation notes that currently act as Pantro's lightweight roadmap and decision trail.

Use these files when you need to understand:

- the current release-readiness priorities
- the intended scope and acceptance criteria for in-flight work
- the implementation decisions recorded during focused delivery sprints

These notes are working documents rather than long-term architecture records. The stable public references remain:

- [README.md](../../README.md) for product scope and install entry points
- [docs/ARCHITECTURE.md](../ARCHITECTURE.md) for service boundaries
- [docs/VERSIONING.md](../VERSIONING.md) and [CHANGELOG.md](../../CHANGELOG.md) for release state
