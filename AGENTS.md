# AGENTS.md

## Pantry repo working rules

- Do not work directly on `main`
- Use short-lived branches: `feature/...`, `fix/...`, `docs/...`, `release/...`
- Open a pull request for all changes
- Keep changes focused and reviewable
- Run relevant tests before considering work complete
- Update public docs when public behaviour changes
- Keep internal planning material out of the public repo
- Keep `private-docs/` local-only and untracked
- Prefer small logical commits
- Do not add SaaS logic to this repo
